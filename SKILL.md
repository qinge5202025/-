## 二、预测方法论 v3（波神核心算法）

采用 **7因子动态权重 + 动态Elo评分 + 庄家动机检测** 的集成模型：

### 七个预测因子

| # | 因子 | 权重 | 说明 |
|:---:|---|---|---|
| ① | 欧赔隐含概率 | 33% | 赔率反推胜平负概率 + 凯利指数价值判断 + 平赔异常检测 |
| ② | 赔率变动方向 | 10% | 初盘→现盘变化，三级阈值分级(强/中/弱信号) |
| ③ | 亚盘盘口分析 | 20% | 盘口深度 + 水位 + 升/降盘变动 + 交叉验证 |
| ④ | 大小球趋势 | 7% | 盘口大小 + 水位 + 战意进球乘数联动 |
| ⑤ | **战意因子** | 11% | 小组排名/积分/出线形势/净胜球/争第一动机/平局倾向度 |
| ⑥ | **动态Elo评分** | 10% | 赛后自动更新，大胜/冷门自适应K值调整 |
| ⑦ | **庄家动机诱盘检测** 🆕 | 9% | 欧赔vs亚盘背离/深盘高水阻盘/浅盘低水诱盘/高抽水风险 |

## 📊 赔率变动分析法（必须使用）

**赔率变动（Odds Movement）** 是预测引擎的核心因子之一，权重 15%（小组赛）~5%（决赛）。

### 原理

博彩公司的初盘→现盘变化反映两件事：
1. **市场资金流向**（大量投注压低赔率的一方）
2. **庄家真实意图**（庄家可能通过变赔引导/阻吓投注）

赔率变动比静态赔率更能提前反映赛果倾向。

### 三档判断规则

| 变动幅度 | 信号强度 | 判定 |
|:---:|:---:|:---|
| 主胜降 >0.05 | 🟢 强 | 主队被资金追捧 → 倾向 **主胜** |
| 客胜降 >0.05 | 🟢 强 | 客队被资金追捧 → 倾向 **客胜** |
| 主客升降 <0.05 | 🟡 弱 | 无明确信号，忽略变动因子 |
| 平赔升 >0.10 | 🔴 | 资金排除平局 → **降低平局概率** |
| 平赔降 >0.08 | 🔵 | 资金防范平局 → **平局概率增加** |

### 实战信号分级

```
示例: 阿根廷 vs 奥地利
  初盘: 主1.58  平3.53  客5.05
  现盘: 主1.36  平4.30  客7.00
  ────────────────────────────
  判定: ✅ 主降0.22 → 强烈主胜信号
        ✅ 平赔升0.77 → 排除平局
  综合: 阿根廷胜概率大增

示例: 瑞士 vs 加拿大
  初盘: 主2.09  平3.20  客3.19
  现盘: 主2.26  平3.15  客2.98
  ────────────────────────────
  判定: ✅ 客降0.21 → 客队被追捧
        ✅ 主升0.17 → 主队被看衰
        ✅ 平赔微降0.05 → 防平信号
  综合: 加拿大不败倾向，平局概率增加
```

### 与战意因子的协同

赔率变动因子必须与 **战意因子**（二、⑤）协同判断：

| 赔率变动 | 战意冲突 | 处理方式 |
|:---|:---|---|
| 主降0.15 | 客队背水一战 | 战意抵消部分赔率信号，置信度降权 |
| 双方保平 | 客降0.20+防平 | **战意+赔率变动双确认→平局** |
| 已出线方(主) | 初盘→现盘无明显变动 | 尊重战意判断，赔率变动因子权重降至最低 |

### Python预测引擎中的实现

```python

# 权重: 小组赛15% | 16强10% | 8强8% | 半决赛5% | 决赛5%

    cd = ec.draw - ei.draw   # 平赔变动

```

### 使用时的注意事项

1. ❌ **初盘缺失时**：不使用赔率变动因子，只依靠其他因子
2. ❌ **变动幅度<0.05**：视为噪音，不生成信号
3. ⚠️ **极端变动**(主胜从1.50→1.05)：警惕大热倒灶，置信度封顶75%
4. ✅ **大小球变动**也用同样的逻辑分析（盘口升降/水位变化）
5. 每场比赛的 `keyFactors` 中必须包含赔率变动判断，如"主胜赔从1.58降至1.36，资金集中追捧"

## 四、赔率分析指南（辅助判断）

赔率数据来自 `scripts/fetch-odds.py`，反映庄家（博彩公司）对比赛结果的判断。**赔率不是概率，而是庄家平衡投注的工具**。

> ⚠️ **必须先阅读三(赔率变动分析法)** — 静态赔率反映当前共识，赔率变动反映趋势变化，两者结合使用效果最佳。

### 赔率核心解读

| 赔率范围 | 含义 |
|:---:|:---|
| < 1.50 | 极度看好，但需警惕大热倒灶 |
| 1.50 ~ 2.00 | 明显优势方 |
| 2.00 ~ 3.00 | 略占优势 / 势均力敌 |
| 3.00 ~ 5.00 | 下风方，存在爆冷可能 |
| > 5.00 | 明显弱势，爆冷难度大 |

### 赔率变动操盘思路

| 变动方向 | 庄家意图 | 赛果暗示 |
|:---|:---|:---|
| 热门方赔率**下调** | 防范热门打出，控制赔付风险 | 看好热门 |
| 热门方赔率**拉升** | 不看好热门，或诱导资金去热门 | ⚠️ 警惕冷门 |
| 平赔**下调** | 防范平局，比赛可能胶着 | 小比分 / 闷平 |
| 平赔**拉升** | 不看好平局 | 倾向分胜负 |
| 冷门方赔率**大幅下调** | 可能有内幕消息/主力伤停未公开 | ⚠️ 爆冷概率上升 |

### 综合分析原则

1. **赔率优先于排名** — 庄家信息比公开新闻更全面
2. **变动比绝对值更重要** — 关注赔率走势而非静止数字
3. **结合基本面** — 赔率异常变动需找基本面原因（伤停、轮换、战意）
4. **不盲从** — 赔率只是辅助，足球有偶然性

## 九、智能预测系统（predict.py）

`scripts/predict.py` 是基于赔率数据的**多因子集成预测引擎**，自动输出比分、赛果、大小球、置信度。

### 算法架构

```
赔率数据 (odds-latest.json)
    │
    ├── 因子1: 赔率隐含概率 (权重40%)
    │    将赔率转换为概率，>55%作为方向信号
    │
    ├── 因子2: 赔率变动方向 (权重25%)
    │    比较初盘/即时盘，识别庄家真实意图
    │
    ├── 因子3: 亚盘盘口水位 (权重25%)
    │    让球深度 + 水位高低 + 盘口变动
    │
    └── 因子4: 大小球盘口 (权重10%)
         盘口大小 + 水位 + 变动方向
              │
              ▼
    集成投票 → 赛果/比分/大小球/置信度
```

### 输出内容

| 输出项 | 说明 |
|:---|:---|
| 赛果预测 | 主胜/平局/客胜 + 置信度百分比 |
| 比分预测 | 最可能比分 + 备选比分 |
| 大小球预测 | 大/小球 + 盘口 + 置信度 |
| 推荐选项 | 高置信度的可参考方向 |
| 因子分析 | 各因子投票详情 |

### 用法

```bash
# 预测最新采集的比赛
python scripts/predict.py

# 指定数据文件
python scripts/predict.py --file data/odds-latest.json

# 输出文件
#   data/predictions-latest.json       → 最新预测
#   data/predictions-YYYYMMDD.json     → 按日期归档
```

## 十、赛后复盘 & 算法自优化（review.py）

`scripts/review.py` 是**闭环学习系统**：记录预测 → 对比赛果 → 统计准确率 → 自动调整算法权重 → 越预测越准。

### 工作流

```
采集赔率 → 自动预测 → 比赛结束 → 输入赛果 → 复盘对比 → 优化权重
                                ↑                              ↓
                                └────── 更准的预测 ←──────────┘
```

### 用法

```bash
# 交互式复盘（比赛结束后，逐一输入比分）
python scripts/review.py

# 查看历史复盘报告
python scripts/review.py --report

# 查看算法优化状态
python scripts/review.py --status

# 执行自动优化
python scripts/review.py --optimize

# 批量导入赛果复盘
python scripts/review.py --auto --input data/actual-results.json
```

### 复盘产出

| 指标 | 说明 |
|:---|:---|
| 赛果准确率 | 胜负预测正确率 |
| 大小球准确率 | 大小球预测正确率 |
| 精确比分命中 | 比分精确命中场次 |
| 置信度分桶 | 高/中/低三档的准确率（好的算法应是高置信准确>低置信） |
| 因子准确率 | 各因子独立准确率（需足够样本） |

### 算法自优化原理

基于复盘数据自动调整 `PREDICTION_WEIGHTS`（predict.py 中的四个因子权重）：

1. 统计每档置信度的实际准确率
2. 对比基线准确率（历史平均）
3. 如果高置信准确率 > 基线 → 加大隐含概率因子权重（算法判断正确）
4. 如果中置信准确率 < 基线 → 降低变动因子权重（过度敏感）
5. 样本 >5 场才触发优化，避免过拟合
6. 优化历史保存在 `data/performance.json`
7. 每次优化后**自动同步权重到 predict.py**

---

## 十一、全自动工作流（pipeline.py）

一键执行完整流程：

```bash
# 完整采集+预测
python scripts/pipeline.py

# 仅预测（用已有数据）
python scripts/pipeline.py --no-fetch

# 全流程：采集→预测→复盘→优化
python scripts/pipeline.py --all
```

### 日常使用建议

| 时机 | 操作 |
|:---|:---|
| 比赛日当天 | `python scripts/pipeline.py` → 获取最新赔率+预测 |
| 比赛结束后 | `python scripts/review.py` → 输入赛果，复盘 |
| 复盘≥5场后 | `python scripts/review.py --optimize` → 自动优化权重 |
| 随时 | `python scripts/review.py --status` → 查看优化状态 |
