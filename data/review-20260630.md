## 二、代码审查发现的问题

### 问题1: 票数(votes)永远为0 🔴

  文件: engine.py → predict_result()
  现象: 所有预测输出的 votes 字段都是 {home: 0, draw: 0, away: 0}
  原因: predict_result() 设置了 value 和 confidence, 但从未填充 votes 字段!
  影响: 失去了"多因子投票明细", 无法追踪每个因子投给了谁

  修复方案: 在 predict_result() 末尾添加:
    ctx.result_prediction['votes'] = {
        'home': <各因子投给主胜的票数>,
        'draw': <各因子投给平局的票数>,
        'away': <各因子投给客胜的票数>,
    }

### 问题2: 置信度校准公式不准确 🔴

  文件: engine.py → predict_result() (约第870行)
  现状公式:
    prob_margin < 0.05 → 33%
    prob_margin < 0.10 → 38%~44%
    prob_margin < 0.20 → 44%~56%
    prob_margin < 0.30 → 56%~66%
    prob_margin > 0.30 → 66%~81%
    + 赔率变动修正 +0.03~0.06

  问题: 这个线性映射不准确。高概率差(>30%)的场次准确率仅50%,
        说明"赔率差大 ≠ 结果确定"的情况很常见 (爆冷/平局)

  修复方案: 
    1) 引入冷门风险修正: 当 Elo 差距大但赔率差也大时, 降低置信
    2) 平局概率 > 25% 的场景, 置信度统一封顶70% 
    3) 加入"战意矛盾"降权: 强队已出线时, 即使赔率差大, 置信度-10%

### 问题3: 战意分析缺失 🟡

  现状: motivation_analysis 始终生成
    "巴西: 小组赛争夺积分"
    "日本: 小组赛争夺积分"
  但实际比赛已是小组赛第三轮以后, 各队战意完全不同!

  例如(以实际小组积分推断):
    巴西(7分,已出线) vs 日本(5分,需取胜才能确保出线)
    → 正确的战意: 巴西"已出线,可能留力", 日本"背水一战"
    → 当前算法一律"小组赛争夺积分", 导致该场景的平/冷判断偏差

  修复方案: 在 data_pipeline.py 或 engine.py 中, 根据 group-standings.json
    的积分自动推算每队的战意类型:
    - 已提前出线 → '已出线'
    - 取胜或打平即可出线 → '保平出线'
    - 必须赢才能出线 → '背水一战'
    - 已被淘汰 → '荣誉之战'

### 问题4: 比赛阶段(stage)未正确更新 🟡

  现状: predictions-latest.json 中 stage 始终为 "小组赛"
  但6月28日起比赛可能是淘汰赛(32强)了

  影响: engine.py 使用了不同的阶段权重:
    小组赛: odds_implied=0.35, elo=0.11
    16强:   odds_implied=0.28, elo=0.30

  淘汰赛用小组赛权重 → Elo评分权重过低, 赔率权重过高!

### 问题5: odds_movement 数据空洞 🟡

  现状: 所有预测的 odds_movement 都是:
    "spf_change": {"home": 0, "draw": 0, "away": 0},
    "movement_strength": "STABLE",
    "signals": ["赔率稳定, 无明显变动"]

  问题: fetch-odds.py 没有采集到初盘数据, 所以无法计算变动。
        没有初盘→现盘的对比, odds_movement 因子形同虚设。

  影响: odds_movement 因子权重 12% (小组赛) 全部浪费。
        真实准确率应该只看其他因子, 即实际只用了 88% 的权重在预测。

### 问题6 | over_under 预测完全不可用 🟡

  over_under 因子 7% 权重, 但 accuracy 仅 40%。
  实际可用只有亚盘(80%)和赔率隐含(70%)。
  建议: 小组赛阶段临时将 over_under 的权重分配给 elo 因子。
